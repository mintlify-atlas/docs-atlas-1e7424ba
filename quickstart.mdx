This guide walks you through finding a skill, loading it into an AI platform, and having your first conversation with a persona. By the end, you will have a working skill running in your preferred AI assistant and a clear sense of how to explore the rest of the catalog.

If you are not sure which skill to try first, start with [Nuwa.skill](/skills/tools) — it is a meta-tool that helps you create custom skills — or pick any skill from the [Business Mindset](/skills/business-mindset) category, which tend to be the most immediately useful for practical reasoning tasks.

## Load your first skill

<Steps>
  <Step title="Browse the catalog">
    Open Skills organizes its skills into eight categories. Browse the category pages to find a skill that fits what you want to do.

    <CardGroup cols={2}>
      <Card title="Workplace & Academia" icon="briefcase" href="/skills/workplace-academia">
        Bosses, colleagues, professors, and HR personas
      </Card>
      <Card title="Business Mindset" icon="chart-line" href="/skills/business-mindset">
        Buffett, Munger, Jobs, Musk, Naval, and more
      </Card>
      <Card title="Relationships" icon="heart" href="/skills/relationships">
        Partners, exes, crushes, and family personas
      </Card>
      <Card title="Tools" icon="wrench" href="/skills/tools">
        Meta-skills for creating and evolving other skills
      </Card>
    </CardGroup>

    Each category page lists every skill in that group with a short description and a link to its GitHub repository.
  </Step>
  <Step title="Open the skill's GitHub repository">
    Click the link for the skill you want. Each skill lives in its own GitHub repository under the [tmstack](https://github.com/tmstack) organization or a linked community repository.

    The repository contains:
    - A `README.md` with an overview of the persona and usage instructions
    - The `.skill` definition file with the full system prompt and persona configuration
    - Sample interactions that show the persona in action

    Read the README before loading the skill. It tells you what the skill is designed for, any platform-specific instructions, and whether the skill supports Chinese, English, or both.
  </Step>
  <Step title="Copy the system prompt">
    Open the `.skill` file in the repository. The file contains the system prompt you will paste into your AI platform. Copy the full contents of the file.

    <Info>
      Some skills include separate Chinese and English versions of the system prompt. Use the version that matches the language you want to converse in.
    </Info>
  </Step>
  <Step title="Load the skill into your AI platform">
    Paste the system prompt into the custom instructions or system prompt field of your AI platform. Open Skills works with any AI assistant that supports custom system prompts, including:

    - **ChatGPT** — Use a Custom GPT or paste into the system prompt via the API
    - **Claude** — Paste into the system prompt field when starting a new conversation, or use the Claude API
    - **Any OpenAI-compatible API** — Set the system prompt in your API call

    Save the configuration and start a new conversation. The AI will now respond as the persona defined in the skill.
  </Step>
  <Step title="Start a conversation">
    Open a new conversation and address the persona naturally. You do not need to use special commands. Simply talk to it as you would talk to the real person.

    A few things to try when getting started:

    - Ask the persona a question in their area of expertise. For Buffett.skill, try: "I'm looking at a software company with strong recurring revenue but no moat. What should I be worried about?"
    - Push back on something the persona says. A well-distilled skill maintains its perspective under pressure rather than immediately agreeing with you.
    - Ask the persona to explain their reasoning. Good skills articulate the mental models behind their responses.
  </Step>
</Steps>

<Warning>
  Skills are AI persona definitions, not verified knowledge sources. A Buffett.skill will reason like Warren Buffett, but it does not have access to his current views or private information. Treat skill output as a thinking tool, not as authoritative advice.
</Warning>
